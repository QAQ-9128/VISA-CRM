# 交接:概览 / Dashboard 页(UX + UI 优化)

## 目标
在现有概览页基础上做**信息架构重排 + 视觉/可读性优化**,**不删任何功能、不改数据来源**。预览:打开 `design_files/preview.html`(默认进概览)。组件见 `crm-layoutA.jsx` 的 `LayoutABody / Checklist / BarA2 / StatA`;沿用现有设计系统(亮蓝主色、圆角大卡、柔和阴影、`StageBadge` 阶段色)。

## About the Design Files
HTML/JSX 参考稿,非上线代码。在现有栈(React 19 + TS + Tailwind v4 + TanStack Query + Supabase)用既有 hooks 复刻。

---

## 要改的 8 点(逐项)

1. **统一"待办"概念**(当前最乱):原来「待办事项=0」却有 8 条手写清单、还有「我的待办(空)」「待办案件」并存,自相矛盾。
   - 把 **「我的待办」+「待办清单」合并成一个组件**(手写任务:回车添加 / 勾选完成 / 删除;见 `Checklist`)。
   - **「待办事项」统计卡数字** = 该清单未完成条数(或系统任务真实数),不要恒为 0。
   - **「待办案件」改名「待办阶段案件」**,副标/标题注明是"处于待办阶段的案件",与个人任务区分。

2. **收敛空状态、平衡密度**:即将到期 / 逾期未付款 / 星标客户 空着时不要各占一张大卡导致下半页大片空白。空态用紧凑占位,或把"提醒/关注"类合并到一个分区。整页上下密度要均衡。

3. **欠款上移、去重**:顶部「未付总额」与原来页尾「欠款总览(按客户)」是同一笔钱。把**客户欠款做成顶部统计卡 + 紧凑的"欠款总览(按客户)"卡上移**到首屏区域;不要页头页尾各放一次同一个 190k。

4. **金额格式统一**:全站统一 `formatMoney`(同一精度 + 千分位),如 `AUD 190,000` / `AUD 190,000.09`;杜绝 `0.00` / `190k` / `190,000.09` 混用。

5. **"未命名"兜底**:客户无姓名时显示案件参考号或「签证类型 · 案件」代替「未命名」。

6. **问候个性化**:补当前用户名(「你好,Amy 👋」)。

7. **月度收款趋势图重做可读性**(见 `BarA2`):**每根柱顶显示数值**、主色加深加粗、**当前月高亮**、非当前月用中度蓝(非极浅)、0 值月也保留可见的最小高度与标注。

8. **「下签」与"在办"区分**:案件阶段分布里,中心大数「在办案件」**只统计进行中**(不含下签);下签作为图例项标注「· 已完成」。卡副标显示「在办 N · 已下签 M」。

## 必须保留的功能(勿删)
4 张统计卡 · 案件阶段分布(环形)· 待办阶段案件列表 · 即将到期提醒 · 月度收款趋势 · 递交进度表 · 我的待办/待办清单(合并后)· 逾期未付款 · 星标客户 · 官方签证处理时间外链 · 欠款总览(按客户)。以上数据全部接现有 hooks。
